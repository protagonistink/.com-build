# Build Fix Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Remove the Pexels plugin and install styled-components so `npm run build` passes cleanly.

**Architecture:** Two code changes (sanity.config.ts + package.json), then a clean reinstall and build verification. No logic changes, no tests required — this is pure dependency hygiene.

**Tech Stack:** Next.js 16, Sanity v5, npm

---

### Task 1: Remove Pexels from sanity.config.ts

**Files:**
- Modify: `sanity.config.ts`

**Step 1: Remove the import**

Delete this line:
```ts
import { pexelsImageAsset } from 'sanity-plugin-asset-source-pexels';
```

**Step 2: Remove the env var reference**

Delete this line:
```ts
const pexelsApiKey = normalizeEnvValue(process.env.NEXT_PUBLIC_PEXELS_API_KEY);
```

**Step 3: Remove the plugin from the plugins array**

Delete this entry (both branches):
```ts
pexelsApiKey ? pexelsImageAsset({ API_KEY: pexelsApiKey }) : pexelsImageAsset({ useProxyClient: true }),
```

**Step 4: Verify the file is clean**

The `plugins` array should now be:
```ts
plugins: [
  structureTool(),
  visionTool(),
  media(),
  unsplashImageAsset(),
  youtubeInput({ apiKey: youtubeApiKey }),
  assist(),
],
```

**Step 5: Commit**

```bash
git add sanity.config.ts
git commit -m "Remove Pexels plugin from Sanity config"
```

### Task 3: Clean install and build

**Step 1: Delete node_modules and package-lock to ensure a clean state**

```bash
rm -rf node_modules package-lock.json
```

**Step 2: Install**

```bash
npm install
```

Expected: installs cleanly with no peer dependency errors or warnings about React version conflicts.

**Step 3: Build**

```bash
npm run build
```

Expected: build succeeds with no `Module not found: Can't resolve 'styled-components'` errors. Output ends with something like:
```
Route (app)                              Size     First Load JS
...
✓ Compiled successfully
```

**Step 4: If build passes, commit the updated lock file**

```bash
git add package-lock.json
git commit -m "Update lockfile after dependency cleanup"
```

**Step 5: If build fails, check the error output**

- If still seeing `styled-components` errors: verify it appears in `node_modules/` and `package.json`
- If new errors appear: read them carefully — do not retry the same commands
